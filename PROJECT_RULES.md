# MK2MD 專案規範

## 核心原則

右鍵選單是完整功能入口。所有使用者能操作的重要功能,都必須能從右鍵選單找到。

上方工具列只允許作為高頻快捷入口。它可以重複放常用功能,但不能取代右鍵選單的功能歸屬。

## UI 歸屬規則

- 新增功能時,先放進右鍵選單或確認右鍵選單已有入口。
- 上方工具列只放高頻快捷操作,不得把右鍵功能搬出來後刪掉右鍵入口。
- 如果上方工具列新增入口,右鍵選單必須仍保留同功能入口。
- 如果要改變功能歸屬,必須先在對話中明確說明「這是歸屬改動」並取得同意。
- 搜尋、檔案、編輯、檢視等一般操作,右鍵選單都必須保留。
- 診斷/自檢屬於維護功能,不要顯示在一般右鍵選單;保留在命令面板即可。

## 修改前檢查

改 UI 前先確認:

- 這次是新增快捷入口,還是改變功能歸屬?
- 右鍵選單是否仍保留一般使用者需要的完整功能?
- 上方工具列是否只承擔快捷入口角色?
- 自檢是否能防止未來又把右鍵入口拿掉?
- README、版號、Changelog 是否同步更新?

## 驗證規則

每次改 UI 後至少驗證:

- `index.html` 內的 app script 可解析。
- 命令面板自檢通過,且診斷/自檢仍可從命令面板找到。
- 右鍵選單自檢通過。
- 上方工具列的快捷入口在右鍵選單仍找得到。
- 一般右鍵選單不顯示診斷/自檢入口。
- GitHub Pages 部署後線上版也通過相同檢查。
