# HTML 區塊範本示範設計

## 目標

在內建 `DEFAULT_MARKDOWN` 範本加入一個安全、容易辨認的 HTML 區塊，讓使用者能直接比較「純文字」與「安全顯示（白名單）」兩種模式。

## 範圍

- 新增一個 `## 🌐 HTML 區塊顯示` 範本段落。
- 使用 `<details open>`、`<summary>`、`<p>`、`<strong>`、`<ul>`、`<li>` 組成示範。
- 純文字模式顯示完整 HTML 原始碼。
- 安全顯示模式呈現可展開、收合的內容，危險標籤與屬性仍由既有白名單移除。
- Markdown 匯出必須逐字保留 HTML 區塊。
- 不加入外部圖片、外部網址、腳本、事件屬性或額外網路請求。

## 位置與說明

範例放在程式碼區塊示範之後、警示框示範之前。說明文字會指引使用者從「右鍵 → 檢視 → HTML 區塊顯示」切換模式。

## 驗證

- 範本統計新增 HTML 區塊數量。
- 範本自測要求至少存在一個 HTML 節點。
- 自測確認匯出 Markdown 仍含 `<details>` 與 `</details>`。
- 執行完整自測，並在本機及 GitHub Pages 驗證 v10.69。

## 不做事項

- 不改變預設純文字模式。
- 不擴大 HTML 白名單。
- 不新增獨立工具列或手機版入口。
- 不把規格文件發布到 GitHub Pages；Pages 仍只發布單檔 `index.html`。
